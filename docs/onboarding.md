# 首次使用引导文档

本文档定义了 Deep Novel System 首次使用时的引导问答流程。

## 引导流程

### 触发条件
- 用户说"我想开始写小说"
- 用户说"启用 deep-novel-system"
- 用户询问如何开始一个新的写作项目

### 问答序列

#### Q1: 项目路径
```
您想把小说项目放在哪里？请提供一个目录路径。

示例：
- Windows: F:\我的小说\项目A
- macOS/Linux: ~/Documents/novel-project
```

**验证**：检查路径是否有效，是否有写入权限

#### Q2: 小说类型
```
您打算写什么类型的小说？

1. 色情/情色
2. 轻小说
3. 现代言情
4. 玄幻/奇幻
5. 悬疑/推理
6. 恐怖/惊悚
7. 其他（请描述）
```

#### Q3: 主要角色
```
请简单描述 2-3 个核心角色：

格式：
- 角色名 | 性别/年龄 | 简短描述 | 与主角关系
```

#### Q4: 核心冲突
```
故事的核心冲突或主题是什么？

例如：
- 背叛与复仇
- 禁忌之恋
- 成长与蜕变
- 权力的游戏
```

#### Q5: 预计篇幅
```
预计的篇幅是？

1. 短篇（< 5万字）
2. 中篇（5-20万字）
3. 长篇（> 20万字）
```

#### Q6: 特殊要求（可选）
```
是否有特殊的写作风格或禁忌词汇要求？

例如：
- 第一人称视角
- 禁止使用某些词汇
- 特定的 NSFW 偏好
```

## 基础文件生成规则

### config/instruction.md
根据小说类型自动选择预设：
- 色情类 → 启用 NSFW 相关配置
- 恐怖类 → 氛围营造优先、心理惊悚、留白艺术
- 轻小说 → 轻量化对话风格
- 其他 → 通用叙事配置

### plans/global_plan.md
基于 Q4（核心冲突）填充 `assets/template/plans/global_plan.md` 骨架的核心字段：

- `## 元信息` → Q2 类型、Q5 篇幅
- `## 一句话梗概 (Logline)` → 从 Q4 提炼
- `## 核心冲突` → Q4 的外部 / 内部冲突
- `## 篇章规划` → 三幕节拍（铺垫 → 激励事件 → 最低谷 → 高潮 → 收尾）
- `## 章节清单` → 至少填 ch01 的一句话概要

### plans/setting/characters.md
基于 Q3（角色描述）填充 `assets/template/plans/setting/characters.md` 骨架里每个角色的：

- 基础信息（Q3 的性别 / 年龄 / 身份）
- 性格核心（从描述推断，三个关键词 + 一句话总结）
- 说话特点 / 动机 (Want vs Need) / 恐惧与软肋 —— 首次未知可留空
- 与主角关系（Q3 提取）
- 发展弧线（起点 → 中段 → 终点，先写起点即可）

### plans/setting/world.md
若 Q2 类型涉及架空 / 奇幻 / 科幻，必须填充世界观关键设定规则。现代言情类可先留空，随写作推进再补。

### plans/chapters/ch01_plan.md
模板已提供章节大纲样板。按 Q3 / Q4 填充 ch01 的：

- 开场钩子、中段节拍、收尾悬念
- 分节计划（每节约 4000 字）
- 本章结束状态（Exit State）
