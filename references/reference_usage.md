# 参考资料使用与风格学习规则 (Reference Usage Rules)

## 目的
确保 AI 在写作前能够“学习”并模仿 `references/` 目录中提供的范文或风格指南，从而保持文风的一致性或达成特定的写作要求，而不是仅凭通用训练数据写作。

## 资料类型
1.  **风格指南 (`style_guide.md`)**: 明确的“条款式”要求（如“多用短句”、“禁止心理描写”）。
2.  **范文/样本 (`sample_text.md`)**: 用于模仿的文本片段。需要 AI 进行隐式学习（Imitation Learning）。
3.  **世界观设定 (`world_bible.md`)**: 事实性参考，用于确保名词和设定准确。

## 学习与预处理流程 (Style Injection)
在正式生成的 **Drafting** 阶段之前，必须执行 **Style Analysis** 步骤：

### 1. 风格提取 (针对范文)
如果提供了范文，模型应在思维链 (Chain of Thought) 中分析以下维度：
-   **词汇密度**: 是华丽辞藻堆砌，还是平实口语化？
-   **句式节奏**: 长短句的分布。是否有大量的短句强调？
-   **叙事视角**: 第一人称的心理活动占比，还是第三人称的客观描述？
-   **感官侧重**: 是否侧重于视觉、听觉或触觉描写？

### 2. 规则对齐 (针对指南)
-   将 `style_guide.md` 中的“强制要求”转换为 System Prompt 中的约束指令。

## 写作中的应用
-   **Imitation (模仿)**: 在生成正文时，尝试复用范文中的特定修辞手法（如：特定的比喻方式）。
-   **Consistency Check (一致性检查)**: 在生成的每段结尾，快速回检是否偏离了参考资料的基调。

## 示例思维链 (Think Process)
> "用户提供了《赛博朋克风格范文》。分析发现：大量使用霓虹灯、雨水、金属质感的词汇；句式破碎；多用日语借词。
> -> **写作策略**: 我在描写新的街道场景时，要加入‘全息广告牌的电流声’和‘酸雨腐蚀的混凝土’细节，避免使用过于古典或田园的形容词。"
